Section 01

From Shor to Steane — the Key Improvement

The Shor code (Tutorial I) was the first proof that quantum error correction exists. It uses 9 qubits to encode 1 logical qubit and achieves distance 3 — correcting any single-qubit error. But 9 qubits is far from optimal. Andrew Steane's 1996 insight was to abandon the concatenated structure entirely and build a quantum code directly from a classical linear code, using a framework called the CSS construction.

The result is the $[[7, 1, 3]]$ Steane code: same distance as Shor, same correction power, but only 7 physical qubits. It is the smallest quantum code with this capability, tight against the quantum Hamming bound. More importantly, it inherits elegant structure from classical coding theory that makes it far easier to analyze, implement, and extend.

Shor Code [[9,1,3]]

9 physical qubits, 8 stabilizers.

Structure: two concatenated repetition codes. Bit-flip and phase-flip protection are completely separate layers.

X and Z stabilizers act on different qubit sets; the code is not symmetric under $H^{\otimes 9}$.

Transversal gates: limited.

Steane Code [[7,1,3]]

7 physical qubits, 6 stabilizers.

Structure: CSS code from the classical $[7,4,3]$ Hamming code. Bit-flip and phase-flip protection arise from the same parity-check matrix.

X and Z stabilizers act on identical qubit supports — the code is self-dual under $H^{\otimes 7}$.

Transversal gates: $X$, $Z$, $H$, $\text{CNOT}$, $S^\dagger$.

The Central Idea

Take a classical linear code $C$ that contains its own dual ($C^\perp \subseteq C$). Build an $X$-type stabilizer group from the rows of $C$'s parity check matrix, and a $Z$-type stabilizer group from the same rows. Because $C$ contains its dual, these two sets of stabilizers automatically commute — giving you a valid quantum code. The correction capability of the quantum code inherits the distance of the classical code.

Section 02

The CSS Construction

Classical codes and their duals

Recall that a classical binary linear $[n, k, d]$ code $C$ is defined by a parity check matrix $H \in \mathbb{F}_2^{(n-k)\times n}$: a binary string $\mathbf{x} \in \mathbb{F}_2^n$ is a codeword if and only if $H\mathbf{x}^\top = \mathbf{0}$. The dual code $C^\perp$ consists of all binary strings orthogonal to every codeword of $C$: $C^\perp = \{\mathbf{y} : \mathbf{x}\cdot\mathbf{y} = 0 \text{ for all } \mathbf{x}\in C\}$.

The CSS construction requires two classical codes $C_1$ and $C_2$ with $C_2^\perp \subseteq C_1$. For the self-dual or weakly self-dual case used by Steane, we take $C_1 = C_2 = C$ and require $C^\perp \subseteq C$. This is equivalent to asking that the rows of $H$ are themselves codewords of $C$, i.e. $H H^\top = 0$ over $\mathbb{F}_2$.

CSS Theorem (Calderbank–Shor–Steane)

Let $C = [n, k, d]$ be a classical binary linear code with $C^\perp \subseteq C$ and parity check matrix $H \in \mathbb{F}_2^{(n-k)\times n}$. Then there exists a quantum code $\mathcal{Q} = [[n,\, 2k - n,\, d]]$ with:

  • $n-k$ $X$-type stabilizer generators, one per row of $H$.
  • $n-k$ $Z$-type stabilizer generators, one per row of $H$.
  • Distance $d$ inherited from the classical minimum distance.

The stabilizers commute because $H H^\top = 0 \pmod{2}$: every $X$-stabilizer shares an even number of qubits with every $Z$-stabilizer, so they commute.

Why the [7,4,3] Hamming code?

The classical binary $[7, 4, 3]$ Hamming code is the perfect single-error-correcting code on 7 bits. Its parity check matrix is:

q₀ q₁ q₂ q₃ q₄ q₅ q₆
H = [ 1 0 1 0 1 0 1 ] ← r₁
[ 0 1 1 0 0 1 1 ] ← r₂
[ 0 0 0 1 1 1 1 ] ← r₃

This matrix has the remarkable property that its three rows are themselves codewords of the Hamming code (each has weight 4, which is ≥ 3). One can verify that $H H^\top = 0 \pmod{2}$, confirming $C^\perp \subseteq C$. Therefore, applying the CSS theorem yields the $[[7, 2\cdot4-7, 3]] = [[7,1,3]]$ Steane code.

Perfect Hamming code → perfect quantum code

The classical $[7,4,3]$ Hamming code saturates the classical Hamming bound. Its quantum descendant, the Steane $[[7,1,3]]$ code, saturates the quantum Hamming bound: $\sum_{j=0}^{1}\binom{7}{j}3^j = 1 + 21 = 22 \leq 2^{7-1} = 64$. The Steane code is the smallest possible quantum code that corrects all single-qubit errors, achievable only because the Hamming code is perfect.

The syndrome decoding miracle

In the classical Hamming code, each column of $H$ is the binary representation of its column index (1 through 7). When a single-bit error occurs at position $k$, the syndrome $H\mathbf{e}_k$ gives exactly the binary representation of $k+1$. This means the syndrome directly encodes the error location: no lookup table is needed beyond a 3-bit binary-to-integer conversion.

Column index: 1 2 3 4 5 6 7
Qubit index: q₀ q₁ q₂ q₃ q₄ q₅ q₆
r₁: 1 0 1 0 1 0 1 (bit 0 of index)
r₂: 0 1 1 0 0 1 1 (bit 1 of index)
r₃: 0 0 0 1 1 1 1 (bit 2 of index)

The quantum Steane code inherits this property for both $X$ and $Z$ errors independently, giving a 3-bit syndrome for each error type that directly decodes to the faulty qubit's index. This is a dramatic improvement over the Shor code's syndrome lookup.

Section 03

Mathematical Formulation

Logical codewords

The logical codewords of the Steane code are superpositions of all elements of the dual code $C^\perp = [7,3,4]$ shifted by a logical representative. Let $\mathbf{c}_i$ denote the $2^3 = 8$ codewords of $C^\perp$. The logical codewords are:

$$|0_L\rangle = \frac{1}{\sqrt{8}} \sum_{\mathbf{v} \in C^\perp} |\mathbf{v}\rangle, \qquad |1_L\rangle = \frac{1}{\sqrt{8}} \sum_{\mathbf{v} \in C^\perp} |\mathbf{v} + \mathbf{t}\rangle$$

where $\mathbf{t}$ is any coset representative of $C / C^\perp$, i.e. any codeword in $C$ that is not in $C^\perp$. Expanding this concretely, the 8 codewords of $C^\perp$ are:

$C^\perp$:   0000000, 1010101, 0110011, 1100110, 0001111, 1011010, 0111100, 1101001

so the logical zero state is:

$$\boxed{|0_L\rangle = \tfrac{1}{\sqrt{8}}\bigl(|0000000\rangle + |1010101\rangle + |0110011\rangle + |1100110\rangle + |0001111\rangle + |1011010\rangle + |0111100\rangle + |1101001\rangle\bigr)}$$

and $|1_L\rangle$ is obtained by flipping all bits in each term (applying $X^{\otimes 7}$):

$$|1_L\rangle = \tfrac{1}{\sqrt{8}}\bigl(|1111111\rangle + |0101010\rangle + |1001100\rangle + |0011001\rangle + |1110000\rangle + |0100101\rangle + |1000011\rangle + |0010110\rangle\bigr)$$
Self-duality under the Hadamard

Because the Steane code is self-dual ($C^\perp \subseteq C$), applying $H^{\otimes 7}$ maps $|0_L\rangle \leftrightarrow |+_L\rangle$ and $|1_L\rangle \leftrightarrow |-_L\rangle$, just as the single-qubit Hadamard maps $|0\rangle \leftrightarrow |+\rangle$. This means $H^{\otimes 7}$ is a transversal logical Hadamard: it implements $\bar{H}$ on the logical qubit. The Shor code, being non-self-dual, cannot do this.

Logical operators

The logical Pauli operators must (i) commute with all stabilizers and (ii) not themselves be stabilizers. The weight-7 representatives are the simplest to state:

$$\bar{X} = X_0 X_1 X_2 X_3 X_4 X_5 X_6, \qquad \bar{Z} = Z_0 Z_1 Z_2 Z_3 Z_4 Z_5 Z_6.$$

But these can be reduced modulo the stabilizer group. The minimum-weight logical operators have weight 3, for example:

$$\bar{X}_{\min} = X_0 X_2 X_4 X_6 \cdot (X\text{-stabilizer})^{-1} \sim X_4 X_5 X_6, \qquad \bar{Z}_{\min} \sim Z_4 Z_5 Z_6.$$

The existence of weight-3 logical operators is equivalent to the code having distance 3: any 3-qubit Pauli operator that commutes with all stabilizers but is not itself a stabilizer constitutes a logical operation — and therefore an uncorrectable 3-qubit error pattern.

Code parameters and the quantum Hamming bound

The Steane code is a $[[7, 1, 3]]$ code: $n = 7$ physical qubits, $k = 1$ logical qubit, distance $d = 3$. It corrects any single-qubit error ($t = 1$).

Quantum Hamming Bound — saturated

The quantum Hamming bound for a non-degenerate $[[n, k, d]]$ code correcting $t$ errors is: $$\sum_{j=0}^{t} \binom{n}{j} 3^j \leq 2^{n-k}.$$

For $t=1$: the number of correctable errors per qubit is 3 (one $X$, one $Y$, one $Z$), plus the identity. For $n=7, k=1$:

$\binom{7}{0}3^0 + \binom{7}{1}3^1 = 1 + 21 = 22 \leq 2^{7-1} = 64$. The bound is satisfied. Moreover, no $[[n,1,3]]$ code exists for $n < 7$, making the Steane code optimal among distance-3 single-logical-qubit codes.

Section 04

Stabilizer Structure

The Steane code has 6 stabilizer generators — 3 of $X$-type (detecting $Z$ errors) and 3 of $Z$-type (detecting $X$ errors) — each derived directly from a row of the Hamming parity check matrix $H$.

Generators from the parity check matrix

The three $X$-type generators are formed by replacing each 1 in a row of $H$ with an $X$ gate on the corresponding qubit (and $I$ on the others). The $Z$-type generators are formed identically using $Z$ gates.

Generator Type Qubits acted on Qiskit string (rightmost = q₀) Detects
$g_1$ X-type $q_0, q_2, q_4, q_6$ XIXIXIX $Z$ errors (phase-flip)
$g_2$ X-type $q_1, q_2, q_5, q_6$ XXIIXXI $Z$ errors (phase-flip)
$g_3$ X-type $q_3, q_4, q_5, q_6$ XXXXIII $Z$ errors (phase-flip)
$g_4$ Z-type $q_0, q_2, q_4, q_6$ ZIZIZIZ $X$ errors (bit-flip)
$g_5$ Z-type $q_1, q_2, q_5, q_6$ ZZIIZZII $X$ errors (bit-flip)
$g_6$ Z-type $q_3, q_4, q_5, q_6$ ZZZZIII $X$ errors (bit-flip)
Self-dual symmetry

Notice that $g_1$ and $g_4$ act on exactly the same set of qubits — only the Pauli type ($X$ vs $Z$) differs. The same holds for $(g_2, g_5)$ and $(g_3, g_6)$. This is the hallmark of a CSS code built from a self-dual classical code. The $X$ and $Z$ stabilizers are isomorphic, which guarantees that the code treats bit-flip and phase-flip errors on equal footing.

Why the stabilizers commute

Two Pauli operators $P$ and $Q$ on $n$ qubits commute if and only if they overlap on an even number of positions where both are non-identity. For the Steane code: any $X$-stabilizer $g_i$ and $Z$-stabilizer $g_j$ overlap on the positions where both the $i$-th and $j$-th rows of $H$ are 1. The overlap count equals the inner product $r_i \cdot r_j$ over $\mathbb{F}_2$. Since $H H^\top = 0 \pmod 2$, every such inner product is zero — meaning the overlap is even, and the operators commute.

H HT mod 2 — must equal 0
r₁·r₁ = 1+0+1+0+1+0+1 = 4 ≡ 0   ✓
r₁·r₂ = 0+0+1+0+0+0+1 = 2 ≡ 0   ✓
r₁·r₃ = 0+0+0+0+1+0+1 = 2 ≡ 0   ✓
r₂·r₂ = 0+1+1+0+0+1+1 = 4 ≡ 0   ✓
r₂·r₃ = 0+0+0+0+0+1+1 = 2 ≡ 0   ✓
r₃·r₃ = 0+0+0+1+1+1+1 = 4 ≡ 0   ✓

Syndrome decoding table

The 3-bit $Z$-syndrome $(s_1, s_2, s_3)$ encodes the location of an $X$ error; the 3-bit $X$-syndrome $(s_4, s_5, s_6)$ encodes the location of a $Z$ error. In both cases the syndrome bits are precisely the binary representation of the faulty qubit's index (1-indexed), so decoding is arithmetic: $k = s_1 \cdot 1 + s_2 \cdot 2 + s_3 \cdot 4 - 1$ for qubit $q_k$.

The correction type ($X$ or $Z$) is determined by which syndrome (Z-stabilizers → $X$ correction; X-stabilizers → $Z$ correction). A $Y = iXZ$ error triggers both syndromes simultaneously, and both are corrected independently — the two correction paths are orthogonal.

Logical operators and the codespace dimension

The codespace is the simultaneous $+1$ eigenspace of all 6 stabilizers. The 6 generators each halve the available Hilbert space dimension, leaving $2^7 / 2^6 = 2$ dimensions — exactly enough for one logical qubit. The logical operators $\bar{X}$ and $\bar{Z}$ commute with all generators (by the same inner-product argument) but anticommute with each other, confirming they implement the algebra of a single qubit.

Transversal gates — a key advantage

A gate is transversal if it can be implemented by applying independent single-qubit (or two-qubit) operations to each physical qubit, without coupling between the code blocks. Transversal gates are desirable because they do not spread errors within a block.

Logical $\bar{X}$

Apply $X$ to every physical qubit: $X^{\otimes 7}$.

Logical $\bar{Z}$

Apply $Z$ to every physical qubit: $Z^{\otimes 7}$.

Logical $\bar{H}$

Apply $H$ to every physical qubit: $H^{\otimes 7}$. Valid because the code is self-dual.

Logical CNOT

Apply CNOT between corresponding physical qubits of two code blocks: $\text{CNOT}^{\otimes 7}$.

Logical $\bar{S}^\dagger$

Apply $S^\dagger$ to every physical qubit. Together with $\bar{H}$, generates the full Clifford group logically.

Non-Clifford gates require magic state injection

Transversal gates only generate the Clifford group. Implementing the $T$ gate (needed for universal quantum computation) requires a non-transversal technique such as magic state distillation or code switching. This is a fundamental limitation of any CSS code, not just Steane's.

Section 05

Circuit Implementation — Qiskit

1. Encoding circuit

The encoding maps $|\psi\rangle|000000\rangle$ to the Steane codeword. The qubit $q_0$ holds the logical input; $q_1\ldots q_6$ are ancillas initialised to $|0\rangle$. The strategy mirrors the CSS structure: Hadamard gates create superpositions encoding the dual code $C^\perp$, and CNOT gates spread the information according to $H$.

  1. Apply $H$ to $q_4$, $q_5$, $q_6$ (the three "check" qubits). This creates $|+\rangle$ superpositions that will carry the dual-code structure.
  2. Spread $q_6$ to $q_0$, $q_2$, $q_4$ via CNOT (from row $r_1$ of $H$): $\mathrm{CX}(q_6, q_0)$, $\mathrm{CX}(q_6, q_2)$.
  3. Spread $q_5$ to $q_1$, $q_2$ via CNOT (from row $r_2$): $\mathrm{CX}(q_5, q_1)$, $\mathrm{CX}(q_5, q_2)$.
  4. Spread $q_4$ to $q_3$ via CNOT (from row $r_3$): $\mathrm{CX}(q_4, q_3)$.
  5. Apply CNOT gates from $q_0$ (the logical qubit) to $q_1$, $q_2$, $q_3$, $q_4$, $q_5$, $q_6$ to spread the logical information across all qubits.
q₀ (logical) ──●──────────────────────●──●──●──●──●──●── │ │ │ │ │ │ │ q₁ ──┼────────────────────⊕──┼──┼──┼──┼──┼── ← spreads from q₅, q₀ │ │ │ │ │ │ q₂ ──┼────────────────────┼──⊕──┼──┼──┼──┼── ← spreads from q₅, q₆, q₀ │ │ │ │ │ q₃ ──┼───────────────────────────⊕──┼──┼──┼── ← spreads from q₄, q₀ │ │ │ │ q₄ ──┼──┤H├──●───────────────────────⊕──┼──┼── ← superposition + q₀ │ │ │ │ q₅ ──┼──┤H├──┼──●──●──────────────────────⊕──┼── ← superposition + q₀ │ │ │ │ q₆ ──┼──┤H├──┼──┼──┼──●──●────────────────────⊕── ← superposition + q₀ │ │ │ │ │ └──┘ └──┘ └── (CNOT targets)
Python · Qiskit — Steane encoding circuit
from qiskit import QuantumCircuit, QuantumRegister, ClassicalRegister
from qiskit.quantum_info import Statevector, SparsePauliOp
from qiskit_aer import AerSimulator
import numpy as np

def steane_encode(qc, d):
    """
    Encode d[0] (logical input qubit) into 7 data qubits d[0]..d[6].
    Assumes d[1]..d[6] are initialised to |0⟩.

    The encoding follows the CSS structure:
      - H gates on d[4], d[5], d[6] create superpositions (dual code C⊥).
      - CNOT gates spread information according to the rows of H_7.
      - CNOT gates from d[0] spread the logical qubit across all qubits.
    """
    # ── Step 1: Superpositions on check qubits ─────────────────
    qc.h(d[4])
    qc.h(d[5])
    qc.h(d[6])

    # ── Step 2: Spread d[6] → d[0], d[2]  (row r₁ of H: 1_0 1_0 1_0 1) ──
    qc.cx(d[6], d[0])
    qc.cx(d[6], d[2])
    qc.cx(d[6], d[4])

    # ── Step 3: Spread d[5] → d[1], d[2]  (row r₂ of H: 0 1 1_0 0 1 1) ──
    qc.cx(d[5], d[1])
    qc.cx(d[5], d[2])

    # ── Step 4: Spread d[4] → d[3]        (row r₃ of H: 0 0 0 1 1 1 1) ──
    qc.cx(d[4], d[3])

    # ── Step 5: Spread logical qubit d[0] across all qubits ────
    for target in [1, 2, 3, 4, 5, 6]:
        qc.cx(d[0], d[target])

# ── Build and draw the encoding circuit ────────────────────────────────
d = QuantumRegister(7, 'd')
qc_enc = QuantumCircuit(d)
steane_encode(qc_enc, d)
qc_enc.draw('mpl')

2. Syndrome measurement

Each of the 6 stabilizers is measured using one ancilla qubit. The technique depends on the stabilizer type:

Z-type stabilizer ($g_4, g_5, g_6$)

Ancilla in $|0\rangle$. Apply CX from each target data qubit to the ancilla. Measure ancilla: outcome $0 \to +1$, outcome $1 \to -1$ (parity violated).

X-type stabilizer ($g_1, g_2, g_3$)

Ancilla in $|+\rangle$ (apply $H$ first). Apply CX from ancilla to each target data qubit. Apply $H$ to ancilla, then measure. Outcome $0 \to +1$, outcome $1 \to -1$.

Python · Qiskit — Syndrome measurement
def measure_z_stabilizer(qc, ancilla, data_qubits, classical_bit, creg):
    """Measure a Z-type stabilizer. Detects X (bit-flip) errors."""
    for q in data_qubits:
        qc.cx(q, ancilla)     # data → ancilla: encodes parity
    qc.measure(ancilla, creg[classical_bit])
    qc.reset(ancilla)

def measure_x_stabilizer(qc, ancilla, data_qubits, classical_bit, creg):
    """Measure an X-type stabilizer. Detects Z (phase-flip) errors."""
    qc.h(ancilla)                # ancilla → |+⟩
    for q in data_qubits:
        qc.cx(ancilla, q)      # ancilla → data: X-basis parity
    qc.h(ancilla)                # rotate back
    qc.measure(ancilla, creg[classical_bit])
    qc.reset(ancilla)

def steane_measure_syndrome(qc, d, a, s):
    """
    Measure all 6 Steane stabilizer generators.

    Syndrome register layout:
      s[0], s[1], s[2]  ← Z-stabilizers g4, g5, g6 (detect X errors)
      s[3], s[4], s[5]  ← X-stabilizers g1, g2, g3 (detect Z errors)

    Ancilla register: a[0]..a[5] (one per stabilizer)
    """
    # ── Z-type stabilizers: detect bit-flips (X errors) ────────
    # g4: Z on q₀, q₂, q₄, q₆  (row r₁ of H)
    measure_z_stabilizer(qc, a[0], [d[0], d[2], d[4], d[6]], 0, s)
    # g5: Z on q₁, q₂, q₅, q₆  (row r₂ of H)
    measure_z_stabilizer(qc, a[1], [d[1], d[2], d[5], d[6]], 1, s)
    # g6: Z on q₃, q₄, q₅, q₆  (row r₃ of H)
    measure_z_stabilizer(qc, a[2], [d[3], d[4], d[5], d[6]], 2, s)

    qc.barrier()

    # ── X-type stabilizers: detect phase-flips (Z errors) ───────
    # g1: X on q₀, q₂, q₄, q₆  (row r₁ of H)
    measure_x_stabilizer(qc, a[3], [d[0], d[2], d[4], d[6]], 3, s)
    # g2: X on q₁, q₂, q₅, q₆  (row r₂ of H)
    measure_x_stabilizer(qc, a[4], [d[1], d[2], d[5], d[6]], 4, s)
    # g3: X on q₃, q₄, q₅, q₆  (row r₃ of H)
    measure_x_stabilizer(qc, a[5], [d[3], d[4], d[5], d[6]], 5, s)

3. Classical decoder

The decoder converts a 6-bit syndrome into a pair of correction masks (one for $X$ errors, one for $Z$ errors). Because the syndrome is a binary index, no lookup table is needed — just integer arithmetic.

Python — Classical decoder
def steane_decode_syndrome(syndrome_bits):
    """
    Decode a 6-bit Steane syndrome into (x_correction, z_correction).

    Parameters
    ----------
    syndrome_bits : list of 6 bits, [s0..s5], each 0 or 1.
        s[0], s[1], s[2]  from Z-stabilizers  → identify X errors
        s[3], s[4], s[5]  from X-stabilizers  → identify Z errors

    Returns
    -------
    x_qubit : int or None  — apply X gate here (corrects bit-flip)
    z_qubit : int or None  — apply Z gate here (corrects phase-flip)

    Decoding formula:
        syndrome value = s[0]*1 + s[1]*2 + s[2]*4
        qubit index    = syndrome value - 1    (0-indexed, -1 means no error)
    """
    s = syndrome_bits

    # ── X-error location from Z-stabilizer syndrome ─────────────
    x_syn = s[0] * 1 + s[1] * 2 + s[2] * 4
    x_qubit = x_syn - 1 if x_syn > 0 else None

    # ── Z-error location from X-stabilizer syndrome ─────────────
    z_syn = s[3] * 1 + s[4] * 2 + s[5] * 4
    z_qubit = z_syn - 1 if z_syn > 0 else None

    return x_qubit, z_qubit   # apply X at x_qubit, Z at z_qubit

4. Full round-trip: Encode → Error → Correct

Python · Qiskit — Full Steane error correction circuit
def steane_full_circuit(error_qubit=None, error_type='X', initial_state='0'):
    """
    Build and simulate a complete Steane error correction round.

    Parameters
    ----------
    error_qubit  : int or None   – which data qubit to corrupt (0–6)
    error_type   : 'X'|'Z'|'Y'  – Pauli error to inject
    initial_state: '0'|'1'|'+'|'-'
    """
    d = QuantumRegister(7,  'd')  # data qubits
    a = QuantumRegister(6,  'a')  # ancilla qubits (one per stabilizer)
    s = ClassicalRegister(6, 's')  # syndrome bits
    qc = QuantumCircuit(d, a, s)

    # ── Prepare logical input ───────────────────────────────────
    if initial_state == '1': qc.x(d[0])
    if initial_state == '+': qc.h(d[0])
    if initial_state == '-': qc.x(d[0]); qc.h(d[0])
    qc.barrier()

    # ── Stage 1: Encode ─────────────────────────────────────────
    steane_encode(qc, d)
    qc.barrier()

    # ── Stage 2: Inject error ────────────────────────────────────
    if error_qubit is not None:
        if 'X' in error_type: qc.x(d[error_qubit])
        if 'Z' in error_type: qc.z(d[error_qubit])
    qc.barrier()

    # ── Stage 3: Measure syndrome ────────────────────────────────
    steane_measure_syndrome(qc, d, a, s)

    # ── Simulate ─────────────────────────────────────────────────
    sim    = AerSimulator()
    result = sim.run(qc, shots=1).result()

    syndrome_str = list(result.get_counts().keys())[0]
    # Qiskit returns classical bits in reversed order — fix it
    syn_bits = [int(b) for b in reversed(syndrome_str)]

    # ── Decode and correct ───────────────────────────────────────
    x_q, z_q = steane_decode_syndrome(syn_bits)
    print(f"Syndrome bits : {syndrome_str}  (reversed: {syn_bits})")
    print(f"X correction  : qubit {x_q}  (apply X gate, or None if no error)")
    print(f"Z correction  : qubit {z_q}  (apply Z gate, or None if no error)")
    return qc, syn_bits, x_q, z_q
Section 06

Error Correction Examples

We walk through three canonical scenarios — a bit-flip, a phase-flip, and a combined $Y$ error — showing exactly which syndrome bits flip and how the Hamming-based decoder identifies the faulty qubit in one arithmetic step.

Example 1 — Bit-Flip ($X$) on qubit $q_3$

Suppose the logical state is $|0_L\rangle$ and qubit $q_3$ (the first qubit governed by row $r_3$ of $H$) undergoes a bit-flip $X_3$. The $Z$-type stabilizers detect this error; the $X$-type stabilizers are unaffected.

Syndrome Analysis — X error on q₃

An $X$ error anticommutes with $Z$ on the same qubit. We check each $Z$-stabilizer:

$g_4 = Z_0Z_2Z_4Z_6$: does not act on $q_3$ → commutes → $s[0] = 0$
$g_5 = Z_1Z_2Z_5Z_6$: does not act on $q_3$ → commutes → $s[1] = 0$
$g_6 = Z_3Z_4Z_5Z_6$: acts on $q_3$ → anticommutes → $s[2] = 1$

$X$ errors commute with all $X$-type stabilizers → $s[3] = s[4] = s[5] = 0$.

Syndrome: 001 000
Z-syndrome value: $0\cdot1 + 0\cdot2 + 1\cdot4 = 4$
Error qubit: $4 - 1 = 3$ → $q_3$. ✓
Correction: Apply $X_3$. No $Z$ correction needed.

Python · Qiskit — Example 1
# Example 1: X error on q₃
qc, syn, x_q, z_q = steane_full_circuit(
    error_qubit=3,
    error_type='X',
    initial_state='0'
)

# Expected output:
# Syndrome bits : 001000   (s[2] = 1)
# X correction  : qubit 3  (syndrome value 4 → index 3)
# Z correction  : qubit None

# Apply correction
if x_q is not None: qc.x(d[x_q])  # undo the bit-flip
if z_q is not None: qc.z(d[z_q])  # undo the phase-flip (not needed here)

Example 2 — Phase-Flip ($Z$) on qubit $q_5$

Now suppose qubit $q_5$ suffers a phase flip $Z_5$. The $X$-type stabilizers detect $Z$ errors; the $Z$-type stabilizers are unaffected (since $Z$ commutes with $Z$).

Syndrome Analysis — Z error on q₅

A $Z$ error anticommutes with $X$ on the same qubit. We check each $X$-stabilizer:

$g_1 = X_0X_2X_4X_6$: does not act on $q_5$ → commutes → $s[3] = 0$
$g_2 = X_1X_2X_5X_6$: acts on $q_5$ → anticommutes → $s[4] = 1$
$g_3 = X_3X_4X_5X_6$: acts on $q_5$ → anticommutes → $s[5] = 1$

$Z$ errors commute with all $Z$-type stabilizers → $s[0] = s[1] = s[2] = 0$.

Syndrome: 000 011
X-syndrome value: $0\cdot1 + 1\cdot2 + 1\cdot4 = 6$
Error qubit: $6 - 1 = 5$ → $q_5$. ✓
Correction: Apply $Z_5$. No $X$ correction needed.

# Example 2: Z error on q₅ — test with superposition to stress-test phase

    error_qubit=5,
    error_type='Z',

)

# Expected output:
# Syndrome bits : 000011   (s[4]=1, s[5]=1)
# X correction  : qubit None
# Z correction  : qubit 5  (syndrome value 6 → index 5)

# The logical |+_L⟩ state is fully recovered after applying Z₅

Example 3 — Combined ($Y$) Error on qubit $q_2$

A $Y = iXZ$ error on qubit $q_2$ triggers both the $Z$-stabilizers (from the $X$ component) and the $X$-stabilizers (from the $Z$ component) simultaneously. Because the two syndromes are completely independent, both errors are identified and corrected separately without interference.

Syndrome Analysis — Y error on q₂

From the $X$ component (detected by Z-stabilizers):

$g_4 = Z_0Z_2Z_4Z_6$: acts on $q_2$ → anticommutes → $s[0] = 1$
$g_5 = Z_1Z_2Z_5Z_6$: acts on $q_2$ → anticommutes → $s[1] = 1$
$g_6 = Z_3Z_4Z_5Z_6$: does not act on $q_2$ → commutes → $s[2] = 0$

From the $Z$ component (detected by X-stabilizers):

$g_1 = X_0X_2X_4X_6$: acts on $q_2$ → anticommutes → $s[3] = 1$
$g_2 = X_1X_2X_5X_6$: acts on $q_2$ → anticommutes → $s[4] = 1$
$g_3 = X_3X_4X_5X_6$: does not act on $q_2$ → commutes → $s[5] = 0$

Syndrome: 110 110
Z-syndrome value: $1\cdot1 + 1\cdot2 + 0\cdot4 = 3$ → error qubit $2$ → apply $X_2$
X-syndrome value: $1\cdot1 + 1\cdot2 + 0\cdot4 = 3$ → error qubit $2$ → apply $Z_2$
Correction: Apply both $X_2$ and $Z_2$ (i.e. $Y_2$, up to global phase). ✓

Python · Qiskit — Example 3
# Example 3: Y error on q₂ — verify both syndromes fire at the same qubit
qc, syn, x_q, z_q = steane_full_circuit(
    error_qubit=2,
    error_type='XZ',   # Y = iXZ; inject both X and Z
    initial_state='1'
)

# Expected output:
# Syndrome bits : 110110   (s[0]=1,s[1]=1, s[3]=1,s[4]=1)
# X correction  : qubit 2  (Z-syndrome value 3 → index 2)
# Z correction  : qubit 2  (X-syndrome value 3 → index 2)

# Both corrections applied to the same qubit:
# X₂ · Z₂ = iY₂ ≡ Y₂ up to global phase → Y error fully reversed.

if x_q is not None: qc.x(d[x_q])
if z_q is not None: qc.z(d[z_q])
✦
Summary — What the Steane Code Achieves

The Steane code demonstrates that quantum error correction and classical coding theory are deeply intertwined. By identifying the right classical code — one that contains its own dual — Steane obtained a quantum code that is simultaneously optimal (smallest possible for its parameters), efficient (decoder runs in constant time via binary arithmetic), and rich with transversal gates (entire Clifford group is transversal).

The key contrast with the Shor code is architectural: the Shor code fights bit-flips and phase-flips with two separate mechanisms bolted together; the Steane code treats them symmetrically from the outset, with $X$ and $Z$ stabilizers sharing the same support and the same decoder.

The logical next step in this series is the surface code — a topological code that sacrifices the Hamming-based decoder elegance in exchange for a much more favourable physical error threshold (~1%), making it the leading candidate for fault-tolerant quantum computation on near-term hardware.

Section 01

The Pauli Basis — Every Error Lives Here

Before we can talk about discretizing errors, we need a language to describe single-qubit operators. The four Pauli matrices — including the identity — form a complete basis for the space of $2\times 2$ complex matrices:

$$ \mathbb{I} = \begin{pmatrix}1&0\\0&1\end{pmatrix}, \quad X = \begin{pmatrix}0&1\\1&0\end{pmatrix}, \quad Y = \begin{pmatrix}0&-i\\i&0\end{pmatrix}, \quad Z = \begin{pmatrix}1&0\\0&-1\end{pmatrix}. $$

This is not merely a convenient choice. It is a theorem: any $2\times 2$ complex matrix $M$ can be uniquely written as

$$M = \alpha\,\mathbb{I} + \beta\,X + \gamma\,Y + \delta\,Z, \qquad \alpha,\beta,\gamma,\delta\in\mathbb{C}.$$
(Pauli expansion)

In particular, every error — whether it is a tiny accidental rotation, a full bit-flip, or a strange channel we have never encountered before — is a matrix, and therefore has a Pauli expansion. The three key Pauli errors have immediate physical interpretations:

Bit-flip — X

Flips $|0\rangle \leftrightarrow |1\rangle$. The quantum analogue of a classical bit error.

Phase-flip — Z

Flips the relative phase: $|0\rangle\mapsto|0\rangle$, $|1\rangle\mapsto-|1\rangle$. Has no classical analogue.

Combined — Y = iXZ

Both a bit-flip and a phase-flip simultaneously. Note the useful identity $Y = iXZ$.

Key identity

The relation $Y = iXZ$ will be crucial later. It means any $Y$ error can be decomposed into an $X$ error followed by a $Z$ error (up to a global phase), so correcting $X$ and $Z$ errors separately is sufficient to correct $Y$ errors as well.

Unitarity constraint on the coefficients

If $U$ is a unitary matrix (i.e. $U^\dagger U = \mathbb{I}$), then its Pauli coefficients $\alpha,\beta,\gamma,\delta$ must satisfy

$$|\alpha|^2 + |\beta|^2 + |\gamma|^2 + |\delta|^2 = 1.$$

This looks exactly like a normalisation condition on a 4-component complex vector. The probabilities $|\alpha|^2, |\beta|^2, |\gamma|^2, |\delta|^2$ will turn out to be exactly the probabilities with which the state collapses to each Pauli-error subspace during syndrome measurement. This is the heart of discretization.

Caution — not every normalised vector gives a unitary

The condition $\sum|\cdot|^2 = 1$ is necessary but not sufficient for $U$ to be unitary. Generating four random complex numbers and normalising their $\ell^2$ norm does not generally yield a unitary. For a proper random unitary one should use Qiskit's random_unitary from qiskit.quantum_info, or the Haar measure. In our code below we use the normalised random approach as a convenient way to produce a generic (non-unitary) single-qubit error channel that still illustrates discretization.

Section 02

Continuous vs. Discrete Errors

A central challenge in quantum error correction is that real physical noise is continuous. A qubit coupling to its environment, a miscalibrated gate, or a stray electromagnetic field will apply some operator $U$ that is not one of the four Pauli operators — it might be an infinitesimally small rotation like $U = e^{i\epsilon Z} = \cos\epsilon\,\mathbb{I} + i\sin\epsilon\, Z$.

At first glance this is catastrophic: there are uncountably many possible errors, so it seems impossible to identify and undo each one exactly. But the Shor code (and quantum error correction in general) sidesteps this problem entirely through a remarkable mechanism.

The core insight

After the encoded state interacts with an arbitrary single-qubit error channel, running the syndrome measurement circuit collapses ("projects") the error into exactly one of four discrete outcomes: no error $(\mathbb{I})$, bit-flip $(X)$, phase-flip $(Z)$, or combined $(Y = iXZ)$. The code never needs to know or undo the continuous error directly — the measurement does the discretization for free.

This is why the Shor code can correct arbitrary single-qubit errors, not just the three Pauli errors. In the following sections we will derive this result step by step and verify it in simulation.

Section 03

Pauli Strings and the Qiskit Convention

For a multi-qubit system we build operators by taking tensor products of single-qubit Pauli operators. A Pauli string is such a tensor product; for the 9-qubit Shor code, a Pauli string acts on all nine qubits simultaneously:

$$P = P_8 \otimes P_7 \otimes \cdots \otimes P_1 \otimes P_0, \qquad P_i \in \{\mathbb{I}, X, Y, Z\}.$$

Little-endian ordering in Qiskit

Qiskit uses little-endian (right-to-left) qubit ordering in Pauli strings. In a string like "ABCDEFGHI", the rightmost character acts on qubit $Q_0$ and the leftmost acts on qubit $Q_8$:

$$\underbrace{A}_{Q_8}\;\underbrace{B}_{Q_7}\;\underbrace{C}_{Q_6}\;\underbrace{D}_{Q_5}\;\underbrace{E}_{Q_4}\;\underbrace{F}_{Q_3}\;\underbrace{G}_{Q_2}\;\underbrace{H}_{Q_1}\;\underbrace{I}_{Q_0}$$
A common source of bugs — stabilizer string ordering

Because of this convention, the stabilizer $Z_0 Z_1$ (acting on the first two physical qubits of block 1) must be written as "IIIIIIIZZ", not "ZZIIIIIII".

The string "ZZIIIIIII" actually encodes $Z_8 Z_7$, acting on block 3. This is a silent bug: the code runs without errors but measures the wrong stabilizer. Both source notebooks contained this mistake.

Python · Qiskit — Imports and Pauli string checker
import numpy as np
import random
from qiskit import QuantumCircuit
from qiskit.quantum_info import Statevector, SparsePauliOp

# ── Pauli string checker ────────────────────────────────────────────────────
# Checks a Pauli string (big-endian notation, e.g. 'IXYZ') and reports
# which qubits carry non-identity operations in Qiskit's little-endian order.
def check_pauli_string(error_string):
    valid_paulis = {'I', 'X', 'Y', 'Z'}
    n = len(error_string)
    for char in error_string:
        if char not in valid_paulis:
            raise ValueError("Invalid character in Pauli string.")
    print(f"Pauli string (big-endian): {error_string}")
    print("─" * 50)
    found = False
    for i, op in enumerate(error_string):
        if op != 'I':
            q_idx = n - 1 - i  # big-endian → Qiskit little-endian
            print(f"  Q_{q_idx} ← {op}")
            found = True
    if not found:
        print("  No errors — all qubits are identity.")

# ── Generate a random Pauli string ──────────────────────────────────────────
def random_pauli_string(n_qubits):
    return ''.join(random.choice(['I','X','Y','Z']) for _ in range(n_qubits))

# Try it:
s = random_pauli_string(9)
check_pauli_string(s)
Section 04

Encoding a Logical Qubit with the Shor Code

The 9-qubit Shor code encodes one logical qubit into 9 physical qubits via a two-layer construction: an outer phase-flip repetition code over three blocks, and an inner bit-flip repetition code within each block.

Logical codewords

$$ |0_L\rangle = \frac{1}{2\sqrt{2}}\bigl(|000\rangle+|111\rangle\bigr)^{\otimes 3} \qquad |1_L\rangle = \frac{1}{2\sqrt{2}}\bigl(|000\rangle-|111\rangle\bigr)^{\otimes 3} $$

A general logical state $|\psi_L\rangle = a\,|0_L\rangle + b\,|1_L\rangle$ is the input to the encoding circuit. Note: do not confuse the logical amplitudes $a, b$ with the error coefficients $\alpha,\beta,\gamma,\delta$ introduced later.

Encoding circuit (Qiskit, little-endian)

The encoding proceeds in three stages. The circuit below encodes $|0_L\rangle$ (starting from $|0\rangle^{\otimes 9}$ with qubit $Q_0$ as the logical input).

Python · Qiskit — Shor encoding circuit
def shor_encode():
    qc = QuantumCircuit(9)

    # Step 1 — outer repetition: spread logical qubit across three blocks
    #          Q0 is the logical qubit; Q3 and Q6 become block anchors.
    qc.cx(0, 3)
    qc.cx(0, 6)

    # Step 2 — Hadamard on each block anchor: converts Z-basis to X-basis
    #          so that bit-flip protection works in the Hadamard (phase) basis.
    qc.h([0, 3, 6])

    # Step 3 — inner repetition within each block
    qc.cx(0, 1); qc.cx(0, 2)   # block 1: Q0, Q1, Q2
    qc.cx(3, 4); qc.cx(3, 5)   # block 2: Q3, Q4, Q5
    qc.cx(6, 7); qc.cx(6, 8)   # block 3: Q6, Q7, Q8

    return qc

circuit = shor_encode()
circuit.draw()

Section 05

Applying a Continuous (Unitary) Error

We now model a generic single-qubit error as an operator that acts on one qubit $Q_k$ while leaving all others untouched:

$$U_k = \alpha\,\mathbb{I}_k + \beta\,X_k + \gamma\,Y_k + \delta\,Z_k$$
(single-qubit error on $Q_k$)

To act on the full 9-qubit state we must tensor-embed $U$ into the 9-qubit Hilbert space:

$$U_k = \underbrace{\mathbb{I} \otimes \cdots \otimes \mathbb{I}}_{n-1-k} \otimes\; U\; \otimes \underbrace{\mathbb{I} \otimes \cdots \otimes \mathbb{I}}_{k}$$

Because of Qiskit's little-endian convention, there is a subtle but critical ordering issue when building this operator programmatically. The correct accumulation uses single.tensor(op) (not op.tensor(single)):

Python · Qiskit — Continuous error operator (corrected)
def random_pauli_unitary():
    """Return a SparsePauliOp of the form αI + βX + γY + δZ with a random,
    normalised coefficient vector.  Note: this is NOT guaranteed to be
    unitary — it is used here as a generic single-qubit error operator."""
    coeffs = np.random.randn(4) + 1j * np.random.randn(4)
    coeffs /= np.linalg.norm(coeffs)          # ensures Σ|.|² = 1
    return SparsePauliOp(['I', 'X', 'Y', 'Z'], coeffs), coeffs

def apply_error(state, coeffs, target_qubit, n=9):
    """Embed the single-qubit error U = αI + βX + γY + δZ on qubit target_qubit
    into the full n-qubit Hilbert space and apply it to state.

    Bug fixed: the original code used op.tensor(single) which placed the error
    on qubit (n-1-target_qubit) instead of target_qubit.  The correct
    accumulation is single.tensor(op)."""
    op = None
    for i in range(n):
        if i == target_qubit:
            single = SparsePauliOp(['I', 'X', 'Y', 'Z'], coeffs)
        else:
            single = SparsePauliOp('I')
        # FIX: single.tensor(op)  ← 'single' acts on higher-indexed qubits
        #      NOT op.tensor(single) which reverses the qubit assignment!
        op = single if op is None else single.tensor(op)
    return state.evolve(op)

# ── Example: apply a random error to qubit Q2 (block 1) ─────────────────────
U, coeffs = random_pauli_unitary()
print("Error operator U =", U)
print(f"\nCoefficients α,β,γ,δ = {coeffs}")
print(f"Sum |.|² = {np.sum(np.abs(coeffs)**2):.6f}  (should be 1)")

# Encode logical |0⟩
qc     = shor_encode()
state  = Statevector.from_instruction(qc)

# Apply the error on Q2 (last qubit of block 1)
errored_state = apply_error(state, coeffs, target_qubit=2)
print("\nError applied to Q2.  State is now a superposition of error sectors.")

After this step the 9-qubit system is in the state

$$U_2 |\psi\rangle = \alpha|\psi\rangle + \beta X_2|\psi\rangle + \gamma Y_2|\psi\rangle + \delta Z_2|\psi\rangle.$$

This is a superposition of four distinct error scenarios. No measurement has been performed yet, so the error is still continuous — all four Pauli components are simultaneously present. The magic happens in the next step.

Section 06

Syndrome Measurement and the Collapse

The Shor code's 8 stabilizer generators each have eigenvalues $\pm 1$. Measuring one of them on the errored state produces a classical bit: $+1$ (no anticommutation) or $-1$ (anticommutation detected). Collectively the 8 syndrome bits identify which qubit was affected and which Pauli error occurred.

We compute the syndrome in two equivalent ways:

  • Expectation value: $\langle \psi' | S | \psi' \rangle \in [-1, +1]$ — gives a continuous indication of how much the stabilizer has been violated. Useful for visualising the effect of partial errors.
  • Projective measurement: samples $\pm 1$ with probabilities $(1 \pm \langle S\rangle)/2$ and collapses the state into the corresponding eigenspace. This is what a real quantum device does.
Python · Qiskit — Syndrome measurement (two modes)
def measure_stabilizer_expectation(state, pauli_string):
    """Return the real expectation value ⟨S⟩ ∈ [-1, +1].
    +1 → stabilizer satisfied (no error of this type)
    -1 → stabilizer violated (error detected)"""
    op = SparsePauliOp(pauli_string)
    return state.expectation_value(op).real

def measure_stabilizer(state, pauli_string):
    """Simulate a projective measurement of the stabilizer.
    Returns +1 or -1, sampled from the Born-rule distribution."""
    val = measure_stabilizer_expectation(state, pauli_string)
    p_plus = (1 + val) / 2
    return np.random.choice([+1, -1], p=[p_plus, 1 - p_plus])

# g7 = X0…X5 → "IIIXXXXXX"   g8 = X3…X8 → "XXXXXXIII"

stabs = {

    "g3 (Z3Z4)"  : "IIIIIZZII",

print("Syndrome (expectation values after error on Q2):")

g1 (Z0Z1) → ⟨S⟩ = +1.0000   (no violation — error beyond g1's reach)
g2 (Z1Z2) → ⟨S⟩ = varies   (depends on β component of error)
g7 (X0-X5) → ⟨S⟩ = varies   (depends on δ/γ component of error)
g8 (X3-X8) → ⟨S⟩ = +1.0000   (Q2 not in support of g8)

Note that $g_1 = Z_0Z_1$ commutes with all Pauli operators on $Q_2$ because it acts on different qubits, so it always returns $+1$. Similarly $g_8$ (which acts on $Q_3,\ldots,Q_8$) commutes with any error on $Q_2$. The only stabilizers that can detect a $Q_2$ error are $g_2$ (catches $X$) and $g_7$ (catches $Z$).

Section 07

The Discretization Theorem — Full Derivation

We now prove why measuring the syndrome collapses the continuous error to a discrete Pauli, following the treatment in the IBM Quantum Learning course.

Step 1 — Error creates a superposition of Pauli sectors

Let $|\psi\rangle$ be the 9-qubit encoding of a logical state, and suppose qubit $Q_k$ undergoes the error $U$. Using $Y_k = iX_kZ_k$, we write:

$$U_k|\psi\rangle = \alpha|\psi\rangle + \beta X_k|\psi\rangle + i\gamma X_kZ_k|\psi\rangle + \delta Z_k|\psi\rangle.$$

Each of the four terms lives in a different eigenspace of the stabilizer generators:

  • $|\psi\rangle$ is the +1 eigenstate of all stabilizers (no error).
  • $X_k|\psi\rangle$ anticommutes with any stabilizer $S$ for which $S$ contains a $Z$ on qubit $k$, flipping those stabilizer eigenvalues to $-1$.
  • $Z_k|\psi\rangle$ anticommutes with any stabilizer containing an $X$ on qubit $k$.
  • $X_kZ_k|\psi\rangle$ combines both effects.

Step 2 — The syndrome register becomes entangled with the error

The syndrome measurement circuit introduces 8 ancilla qubits. Just before the ancilla measurements, the joint state of (ancillas $\otimes$ data) takes the form:

$$ \alpha\,|\mathbb{I}\text{-synd}\rangle \otimes |\psi\rangle + \beta\,|X_k\text{-synd}\rangle \otimes X_k|\psi\rangle + i\gamma\,|X_kZ_k\text{-synd}\rangle \otimes X_kZ_k|\psi\rangle + \delta\,|Z_k\text{-synd}\rangle \otimes Z_k|\psi\rangle $$

where $|\mathbb{I}\text{-synd}\rangle$, $|X_k\text{-synd}\rangle$, etc. are distinct 8-qubit computational basis states determined by which stabilizers anticommute with each error. Crucially, these four syndrome states are mutually orthogonal because distinct errors produce distinct syndromes.

Step 3 — Measurement collapses to one sector

When we measure the 8 ancilla qubits, the joint state collapses to exactly one of the four terms, with probabilities:

$$ P(\mathbb{I}) = |\alpha|^2, \quad P(X_k) = |\beta|^2, \quad P(X_kZ_k) = |\gamma|^2, \quad P(Z_k) = |\delta|^2. $$

After measurement and classical correction (applying the appropriate Pauli inverse), the data qubits are returned to $|\psi\rangle$ regardless of which outcome occurred. The full post-correction state, expressed as a density matrix, is:

$$\xi \otimes |\psi\rangle\langle\psi|,$$

where $\xi$ describes the now-classical syndrome register and carries no information about the logical state. This is a product state — the syndrome and data are fully decoupled.

Why this is remarkable

The original continuous error $U = \alpha\mathbb{I}+\beta X+\gamma Y+\delta Z$ may have had infinitely many possible forms. After the syndrome measurement, we only ever need to correct one of four Pauli errors, and we know exactly which one from the syndrome bits. The randomness generated by the error is absorbed into the ancilla qubits, which can be reset and reused.

This is why quantum error correction works: it is not that we reverse an arbitrary continuous transformation. It is that the measurement itself discretizes the error, and we only ever undo a discrete Pauli.

Section 08

Beyond Unitaries: Arbitrary Qubit Channels

So far we have considered unitary errors. Real noise is often non-unitary: dephasing, depolarization, amplitude damping, reset errors. We model such noise with a quantum channel $\Phi$, described by its Kraus decomposition:

$$\Phi(\rho) = \sum_j A_j \rho A_j^\dagger, \qquad \sum_j A_j^\dagger A_j = \mathbb{I}.$$

Each Kraus operator $A_j$ is a $2\times 2$ matrix, so it too has a Pauli expansion:

$$A_j = \alpha_j\mathbb{I} + \beta_j X + \gamma_j Y + \delta_j Z.$$

Running the syndrome measurement and correction on the channel output yields, by the same argument as before:

$$\xi \otimes |\psi\rangle\langle\psi|, \qquad \xi = \sum_j \Bigl(|\alpha_j|^2 \Pi_\mathbb{I} + |\beta_j|^2 \Pi_{X_k} + |\gamma_j|^2 \Pi_{X_kZ_k} + |\delta_j|^2 \Pi_{Z_k}\Bigr),$$

where $\Pi_E$ denotes the projector onto the syndrome register's state corresponding to error $E$. The conclusion is identical to the unitary case: the logical state $|\psi\rangle\langle\psi|$ is fully restored.

Depolarizing channel — a concrete example

The depolarizing channel with error rate $p$ is: $$\Phi(\rho) = (1-p)\rho + \frac{p}{3}(X\rho X + Y\rho Y + Z\rho Z).$$ Its Kraus operators are $A_0 = \sqrt{1-p}\,\mathbb{I}$, $A_1 = \sqrt{p/3}\,X$, $A_2 = \sqrt{p/3}\,Y$, $A_3 = \sqrt{p/3}\,Z$.

After syndrome measurement and correction, the state is recovered with probability 1 (assuming $p$ is small enough that only single-qubit errors occur). The error rate $p$ only affects the probability distribution over the syndrome outcomes — not the ability to correct.

Generalisation to multi-qubit codes

The same principle extends to codes that protect against errors on multiple qubits. In that case, errors on $t$ qubits are expressed as tensor products of Paulis, and the syndrome specifies which multi-qubit Pauli correction to apply. The syndrome measurement still discretizes the error — it now projects onto a larger but still finite set of Pauli strings.

Section 09

Full Demo: Putting It All Together

We now combine all the pieces into a single runnable demonstration. The workflow is:

  1. Generate a random normalised Pauli error operator $U = \alpha I + \beta X + \gamma Y + \delta Z$.
  2. Encode $|0_L\rangle$ with the Shor code.
  3. Apply the continuous error to a chosen target qubit.
  4. Measure all 8 stabilizer generators (expectation values and projective).
  5. Identify which error sector the syndrome corresponds to.
  6. Observe that the syndrome uniquely identifies the Pauli component — discretization.
Python · Qiskit — Complete discretization demo
#!/usr/bin/env python3
# Discretization of Errors — Shor Code Demo
# All bugs from the original notebooks have been corrected.
# Requires: qiskit, qiskit-aer, numpy

import numpy as np
import random
from qiskit import QuantumCircuit
from qiskit.quantum_info import Statevector, SparsePauliOp

# ═══ 1. Pauli string utilities ═══════════════════════════════════════════════

def random_pauli_string(n):
    return ''.join(random.choice(['I','X','Y','Z']) for _ in range(n))

def check_pauli_string(s):
    n = len(s)
    print(f"Pauli string '{s}' (big-endian, rightmost = Q0):")
    for i, p in enumerate(s):
        if p != 'I':
            print(f"  Q_{n-1-i} ← {p}")

# ═══ 2. Error operator ═══════════════════════════════════════════════════════

def random_error_operator():
    """Random αI + βX + γY + δZ with Σ|.|² = 1 (illustrative, not unitary)."""
    c = np.random.randn(4) + 1j * np.random.randn(4)
    c /= np.linalg.norm(c)
    return c

def apply_error(state, coeffs, target, n=9):
    """Embed error on qubit `target`.  Uses single.tensor(op) — see §5."""
    op = None
    for i in range(n):
        s = SparsePauliOp(['I','X','Y','Z'], coeffs) if i == target else SparsePauliOp('I')
        op = s if op is None else s.tensor(op)
    return state.evolve(op)

# ═══ 3. Shor encoding ════════════════════════════════════════════════════════

def shor_encode():
    qc = QuantumCircuit(9)
    qc.cx(0,3); qc.cx(0,6)
    qc.h([0,3,6])
    qc.cx(0,1); qc.cx(0,2)
    qc.cx(3,4); qc.cx(3,5)
    qc.cx(6,7); qc.cx(6,8)
    return qc

# ═══ 4. Syndrome measurement ══════════════════════════════════════════════════

# Shor code stabilizers in Qiskit little-endian notation (rightmost char = Q0)
# Z-type (bit-flip detection within blocks)
STABS_Z = {
    "g1 Z0Z1" : "IIIIIIIZZ",  # block 1
    "g2 Z1Z2" : "IIIIIIZZI",  # block 1
    "g3 Z3Z4" : "IIIIIZZII",  # block 2
    "g4 Z4Z5" : "IIIIZZIII",  # block 2
    "g5 Z6Z7" : "IIIZZIIIII"[1:],  # block 3  →  "IIZZIIIII" trimmed
    "g6 Z7Z8" : "ZZIIIIIII",  # block 3
}
# X-type (phase-flip detection across blocks)
STABS_X = {
    "g7 X0-X5" : "IIIXXXXXX",  # Q0..Q5
    "g8 X3-X8" : "XXXXXXIII",  # Q3..Q8
}

def read_syndrome(state):
    """Return expectation values for all 8 Shor stabilizers."""
    results = {}
    for name, s in {**STABS_Z, **STABS_X}.items():
        results[name] = state.expectation_value(SparsePauliOp(s)).real
    return results

def infer_error(syndrome):
    """Heuristic: identify the dominant error from the syndrome."""
    z_violated = [k for k, v in syndrome.items() if ' Z' in k and v < 0]
    x_violated = [k for k, v in syndrome.items() if ' X' in k and v < 0]
    if     z_violated and     x_violated: return "Y (bit+phase flip)"
    elif   z_violated and not x_violated: return "X (bit-flip)"
    elif not z_violated and   x_violated: return "Z (phase-flip)"
    else                                  : return "I (no error)"

def discretization_demo(target_qubit=2, seed=None):
    if seed is not None:
        np.random.seed(seed)

    # Step 1 — random continuous error
    coeffs = random_error_operator()
    alpha, beta, gamma, delta = coeffs
    print("═" * 60)
    print(f"Error on Q{target_qubit}: U = αI + βX + γY + δZ")
    print(f"  α = {alpha:.4f}  |α|² = {abs(alpha)**2:.4f}")
    print(f"  β = {beta:.4f}  |β|² = {abs(beta)**2:.4f}")
    print(f"  γ = {gamma:.4f}  |γ|² = {abs(gamma)**2:.4f}")
    print(f"  δ = {delta:.4f}  |δ|² = {abs(delta)**2:.4f}")

    # Step 2 — encode and apply error
    state = Statevector.from_instruction(shor_encode())
    errored = apply_error(state, coeffs, target_qubit)

    # Step 3 — read syndrome
    syn = read_syndrome(errored)
    print("\nSyndrome (expectation values, +1 = no violation):")
    for name, val in syn.items():
        bar = "✗" if val < 0 else "✓"
        print(f"  {bar} {name:12s}  ⟨S⟩ = {val:+.4f}")

    # Step 4 — infer which Pauli error the syndrome points to
    inferred = infer_error(syn)
    print(f"\nInferred error type: {inferred}")
    print("(The measurement has discretized the continuous error!)")
    print("═" * 60)

discretization_demo(target_qubit=2, seed=42)

Summary — What the Discretization Demo Shows

For each run: a different continuous error (random $\alpha,\beta,\gamma,\delta$) is applied. Yet in every case, the syndrome expectation values fall into one of four clear patterns, corresponding to an $\mathbb{I}$, $X$, $Y$, or $Z$ error on the target qubit. The dominant Pauli component (largest $|\cdot|^2$) is the most probable syndrome outcome.

This is discretization in action: the measurement forces the error into one of four discrete categories, and the appropriate Pauli correction fully restores the logical state. No knowledge of $\alpha,\beta,\gamma,\delta$ is needed at correction time.
